# 0.1.0b6 — openaca.core consumption facade + OSV-native source queries

Backfilled retroactively; shipped to PyPI on 2026-06-03.

## Highlights

- **`openaca.core` consumption facade.** A public, typed entry point for programs that need to consume OpenACA outputs (BOMs, findings, posture results) without depending on internal scanner modules. Ships with `py.typed` so downstream consumers get full type hints. (ADR-0028, plan 026)
- **OSV-native source queries.** Vulnerability matching now uses OSV's native source queries for git-tracked components; `git_ref` is preserved through BOM round-trips so `scan bom` produces the same OSV results as the original scan. Documented that OSV GIT version queries match tags only, not branches.
- **GitHub MCP source preservation.** GitHub MCP source refs (paths, subdirectories, unversioned refs, case-insensitive identities) are preserved end-to-end through scan → BOM → Fleet upload, fixing several round-trip drift cases.
- **Docker run parsing.** `--net`, `--network-alias`, `--label-file`, `--userns`, resource/runtime flags, and `docker container run` form are now recognized as value-taking flags; parsing handles label-file and userns before image, and global options before the run subcommand. Closes a long tail of misclassified Docker MCP launchers.
- **Identity correctness.** Uppercase git commit SHAs now register as immutable pins; GitHub repo casing is normalized in OSV matching so `Octocat/Foo` and `octocat/foo` collapse to the same identity.
- **BOM round-trip helpers.** `bom_components_from_cyclonedx` pairs refs with their `bom-ref` for cleaner downstream consumption; the public `stamp_osv_query_provenance` helper is now part of the API surface.

## Install

`uv tool install openaca==0.1.0b6` or `pip install openaca==0.1.0b6`.

## Compatibility

Pre-V0 — no back-compat hedging. `openaca.core` is a new public surface; existing internal imports continue to work but consumers should migrate to the facade.
