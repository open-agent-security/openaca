# 0.1.0b8 — dependency resolution fix for fresh installs

## Highlights

- **Fresh installs no longer pull dependency pre-releases.** The install
  scripts passed `--prerelease allow`, which uv applies to the entire
  resolution — so fresh 0.1.0b7 installs resolved httpx `1.0.dev3` (and
  pydantic `2.14.0a1`), and `openaca remote sync` crashed at startup
  (`httpx.Client() got an unexpected keyword argument 'timeout'`). Scan
  commands were unaffected. The scripts now use
  `--prerelease=explicit "openaca>=0.1.0b0"`, which allows openaca's own
  betas while keeping every dependency on stable releases, and the
  package metadata caps `httpx<1.0.dev0` and `pydantic<3` as
  defense-in-depth for `pip --pre` installs. Re-running the install
  script (or an MDM re-push) repairs an affected endpoint in place.
- **Smoke-install CI job.** The dev environment installs from the
  committed lockfile, so it never exercised the fresh-resolution path
  users actually hit. CI now builds the wheel, installs it with deps
  resolved fresh, and runs the user-facing commands (`--version`,
  `scan repo`, `bom repo`, `remote configure` + `sync`) — a
  dependency-induced runtime breakage fails CI instead of a user's
  shell.
- **Masked tokens show their last 4.** `remote configure` printed every
  token as bare `ot_...`; it now prints `ot_...WXYZ` — the same last-4
  the backend stores for display — so users with several tokens can tell
  which one is configured. Short or unknown-shaped secrets stay fully
  masked.

## Install

`uv tool install openaca==0.1.0b8` or `pip install openaca==0.1.0b8`.

## Compatibility

Pre-V0 — no back-compat hedging.

- Installs created by an earlier `install.sh`/MDM script run may carry a
  broken httpx pre-release; re-running the installer upgrades and
  repairs them.
- Avoid `uv … --prerelease allow` when installing openaca manually — uv
  resolves openaca's betas without any flag; pip still needs `--pre`.
