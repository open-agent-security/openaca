# 0.1.0b7 — `openaca remote` CLI + agent-graph occurrence identity

## Highlights

- **`openaca fleet` is now `openaca remote`.** The opt-in upload CLI is
  renamed as a clean break — `openaca remote configure` / `openaca remote
  sync endpoint` replace the `fleet` subcommands, and the token can be
  supplied via `OPENACA_REMOTE_TOKEN` instead of argv. Users upgrading from
  a prior `fleet` build must reconfigure: copy relevant values from
  `~/.config/openaca/fleet.toml` to `~/.config/openaca/remote.toml`, and
  re-run the deploy script to install the new `com.openaca.remote`
  LaunchAgent (the old `fleet` LaunchAgent is not removed automatically).
- **Agent-graph occurrence identity.** Components now carry a stable
  occurrence identity (`mcp-server/<name>`) separated from the coordinates
  used for advisory matching, and findings surface the external
  `match_coordinate` in their source output — so "what is this component"
  and "what did it match against" no longer share one overloaded string.
  (ADR-0029, plan 029)
- **Unpinned MCP launches are now matched against OSV.** Stdio MCP servers
  launched as unpinned packages (`npx`, `uvx`, `uv tool run`) are inferred
  from launcher argv — across option terminators, `--package` flags, short
  Python flags, and other launcher-flag variations — gated on MCP launch
  provenance, and queried against OSV like any other component.
- **Remote-upload privacy hardening.** Uploads now redact endpoint-private
  data before crossing the network boundary: absolute filesystem paths are
  relativized (down to basenames when no known root applies, including
  Windows shapes and paths embedded in JSON property values or component
  identities), URLs are reduced to bare hosts, and credentials in URL
  userinfo are stripped — with the upload contract rejecting any leftovers
  as defense-in-depth.
- **MDM install scripts always upgrade.** The Jamf / Kandji / Intune
  deploy scripts now run `uv tool install --upgrade --prerelease allow`,
  so re-running them advances an endpoint to the latest release instead of
  keeping a stale build, and they pass the remote token via environment
  rather than process argv.

## Install

`uv tool install openaca==0.1.0b7` or `pip install openaca==0.1.0b7`.

## Compatibility

Pre-V0 — no back-compat hedging.

- `openaca fleet` no longer exists; scripts and docs must use
  `openaca remote`. Config migration from `fleet.toml` is **manual** —
  copy values to `remote.toml` and re-run the deploy script.
- BOM identity shapes changed under ADR-0029: consumers reading
  `openaca:identity` should expect `mcp-server/<name>` occurrence
  identities, with match coordinates carried separately.
- Remote uploads redact absolute paths and URL paths/userinfo; backends
  validating uploads see relativized paths and bare-host URLs.
