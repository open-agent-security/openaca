# 0.1.0b4 — endpoint posture + remote MCP inventory + stdio safety

Backfilled retroactively; shipped to PyPI on 2026-05-20.

## Highlights

- **Endpoint posture rules.** New checks flag endpoint URL overrides and MCP `autoApprove` declarations. Findings are attributed to the scope that actually owns each key (user / project / launcher), so noise stays scoped to the file that needs the fix.
- **Remote MCP inventory.** Repo and endpoint scans now inventory remote MCP servers via the `mcp-remote` identity scheme, and surface them in the BOM alongside stdio servers.
- **Stdio MCP rendering safety.** Stdio MCP leaf labels strip CLI flags, drop short-flag values, treat `--` as a passthrough boundary, and redact URL credentials before rendering — preventing secrets in CLI tokens / env-style flags from leaking into BOMs or scan output. Render paths also strip query/fragment from remote MCP URL labels.
- **Source vs. component-type split.** Source ecosystems and component types are now separate concerns; matcher derives `component_type` from legacy ecosystem when `extra` is absent, and plugin component identity is canonicalized to the version-less form so marketplace-qualified refs match correctly.
- **Skill provenance recovery.** Skills whose source provenance was lost during scan now have it reconstructed from lockfiles.
- **Robustness.** `UnicodeDecodeError` from settings and MCP manifest collectors no longer aborts the scan; pre-release component-ecosystem hacks were removed.

## Install

`uv tool install openaca==0.1.0b4` or `pip install openaca==0.1.0b4`.

## Compatibility

Pre-V0 beta — no back-compat hedging. Stdio MCP leaf-label format changed (now strips flags + arguments by default); downstream consumers of those labels should re-check against this release.
