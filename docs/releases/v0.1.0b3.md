# 0.1.0b3 — endpoint scan --project opt-in + Trusted Publishing

Backfilled retroactively; shipped to PyPI on 2026-05-19.

## Highlights

- **`scan endpoint --project` is opt-in.** The `--project` flag for endpoint scans no longer defaults to the current working directory; an unconditional reminder note in the scan output points users to it when project-local agent configuration is relevant.
- **Release-tag safety.** The publish workflow now requires release tags to be cut from `main` before PyPI publish can proceed, eliminating a class of "tagged the wrong branch" mistakes.
- **PyPI Trusted Publishing.** OIDC-based publishing replaces the long-lived PyPI API token; see `docs/pypi-publishing.md` for the setup.
- **Vocabulary.** Internal/external docs retire "agent stack" in favor of "Agent Composition Analysis" / "agent component" to match the framing the BOM artifact established.

## Install

`uv tool install openaca==0.1.0b3` or `pip install openaca==0.1.0b3`.

## Compatibility

Pre-V0 beta — no back-compat hedging. `scan endpoint` now requires `--project` to include project-local config; previously it was inferred from cwd.
