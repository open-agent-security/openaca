## Context

What problem are we solving? What forced this decision now? Include
the specific incident, log, or review feedback if applicable —
future-you needs to know what triggered this so they can judge
whether the trigger still holds.

## Decision

The choice we made, in present tense. One paragraph, no hedging.

## Alternatives considered

- **Alternative A**: brief description — rejected because [reason].
- **Alternative B**: brief description — rejected because [reason].

The rejected alternatives are the most important part of an ADR.
This is what stops a future agent (or external reviewer) from
re-suggesting the same wrong thing.

## Consequences

What this enables. What it costs. What to watch for. Be honest
about the downsides — pretending there are none is how an ADR
becomes the thing a future ADR has to supersede.

## When to revisit

The conditions under which this decision should be reconsidered.
Examples: "if we add a Java-only ecosystem", "if Stage 2 reliability
drops below X%", "if we ever need to support self-hosted runners".
If you can't think of any conditions, the decision is probably more
permanent than you think — say so.
