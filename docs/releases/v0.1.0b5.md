# 0.1.0b5 — Agent BOM as first-class artifact + Claude Code plugin scanning

Notes mirror the GitHub Release that was created manually at tag time; checked in here so the convention applies retroactively.

## Highlights

- **Agent BOM as scan composition boundary.** The scanner now emits an Agent BOM (CycloneDX 1.7 + `openaca:` extensions) as the canonical composition artifact; matching runs against the BOM, findings reference BOM component identities. `openaca scan bom <file>` consumes BOMs as input. (ADR-0022)
- **Claude Code plugin scanning.** Repo mode now traverses bundled plugin trees and surfaces skills, commands, hooks, and agents declared by installed plugins. Symlink escape and nested-plugin cross-contamination are blocked at every walk surface.
- **Bug fixes.** PURL qualifier parsing, non-UTF-8 scan input, malformed JSON handling in `scan bom`, npx URL-style spec edge cases, parse-warning surfacing in `bom repo`.

## Install

`uv tool install openaca==0.1.0b5` or `pip install openaca==0.1.0b5`.

## Compatibility

Pre-V0 — no back-compat hedging. The Agent BOM artifact replaces ad hoc scan composition; existing CLI commands continue to work but `scan bom` is new.
