# 0.1.0 — first public release

OpenACA is now public. `0.1.0` is the first stable release — the same
V0 surface the beta series (`0.1.0b1`–`0.1.0b8`) converged on, promoted
out of pre-release so `pip install openaca` and `uv tool install openaca`
work without pre-release flags.

## What OpenACA does

OpenACA is **Agent Composition Analysis (ACA)** — the agent-stack
analogue of SCA. It resolves the plugins, MCP servers, skills, hooks,
and dependencies an AI agent pulls in into stable component identities,
builds the composition graph, and matches it against known security
advisories. Your dependency scanner reads `package.json` and lockfiles;
OpenACA reads the manifests agents actually install from
(`mcp.json`, `.claude-plugin/plugin.json`, `.claude/settings.json`, …).

## Highlights

- **Two scan modes.** `openaca scan repo` answers "what agent components
  are declared in this repository?" (CI gate / PR check); `openaca scan
  endpoint` answers "what's installed on this machine right now?"
  (laptop / runner / MDM sweep). Same advisory matching in both.
- **Identity resolution + composition graph.** Components without package
  coordinates (a local stdio MCP, a skill at a git subpath, an unpinned
  `npx` launch) resolve to stable identities, and findings trace back
  through the graph to the plugin or skill that introduced them.
- **Advisory intelligence on an open substrate.** Versioned components
  are matched against OSV.dev (GHSA / CVE / PYSEC / MAL) at scan time and
  enriched with OpenACA's bundled agent-context overlays — agent-specific
  impact, taxonomy (OWASP Agentic / MCP Top 10, MITRE ATLAS), and
  evidence level. OpenACA builds on upstream advisory IDs rather than
  minting its own.
- **Agent BOM.** `openaca bom repo` emits a CycloneDX Agent BOM; the
  format is consumable downstream and re-matchable as new advisories land.
- **Configuration-hygiene posture** (`--include-posture`, opt-in):
  unpinned installs, `http://` MCP endpoints, Claude API endpoint
  overrides, MCP auto-approval. Signal, not a gate — never affects exit
  codes.
- **CI-native.** SARIF v2.1.0 output, GitHub workflow annotations
  (auto-selected under `GITHUB_ACTIONS`), and a reference Action at
  `open-agent-security/openaca@v1`.
- **Claude Code plugin.** Four slash commands (`/openaca:scan`, `:bom`,
  `:explain`, `:triage`) for staying inside the agent.
- **Local-first.** The scanner reads manifests locally and queries
  OSV.dev for versioned refs; it does not upload your source.

## Install

`uv tool install openaca` or `pip install openaca`. Or:
`curl -fsSL https://openaca.dev/install.sh | sh`.

## Compatibility

V0, early and evolving. The Agent BOM format and component identities
are pre-1.0 and may change before a stable schema release. Scope is
honest about its edges: declared-manifest (not runtime) analysis,
advisory matching only for components with a matchable coordinate, and
Claude-family hosts first — see the README's Limitations section.
